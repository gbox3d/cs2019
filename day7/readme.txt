교재 255p 배열

배열의 선언
=>5명의 파티원의 골드를 관리하려면 
=>int gold1,gold2,gold3,gold4,gold5;
=>int gold[5];
=> 둘중에 어느것이 나아 보이나요?


배열의 사용
=> 기억공간의사용 => 257p 그림 참고.
=> gold[n] => n (0~4)=> 첨자는 0부터 
=> 선언해준 크기 - 1  까지

배열의 초기화
=> int gold[5] = {120,130,100,110,90}

배열 크기 구하기
=> sizeof(gold)/sizeof(int)
=> 배열의전체크기 / 배열의 데이터형식의크기 = 배열의갯수

배열을 안다는것 => 자료구조,알고리즘 준비단계
=> 1.메모리가 필요한시점에 필요한 만큼 확보
=> 2.잘사용하고(알고리즘)
=> 3.다시 잘 되돌려주는것(가비지 컬랙션)

메모리관리 프로그램
=> -1 데이터의끝
=> -1,0,34,2423 => 데이터 없음
=> 3,-1,0,0,0,2312,99 => 데이터 1개 => 3

=>인덱스 변수로 마지막 데이터위치지정 => nLastIndex
=> -1 =>데이터 없음

=> pop => 데이터를 맨뒤에서 제거
=>3,4,5 => 3,4

=> deleteData => 데이터를 맨 앞에서 제거 
=> 3,4,5 => 4,5

=> push => 데이터를 맨뒤에 추가 
=> 3,4,5 => 7,push => 3,4,5,7

=> insert =>데이터를 맨 앞에서 추가
=> 3,4,5 => 7,insert => 7,3,4,5
